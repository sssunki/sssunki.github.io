Android 播放器实现：音视频基础

发表于2020-11-19|更新于2020-12-12
|阅读量:

开坑，先了解基本概念

Android NDK r17c

CMake 3.10.2

FFmpeg 4.1.6

几个结构体：

AVFormatContext：解封装功能的结构体，文件名，流，时长，bit rate等信息。

这个东西是对于整个一个视频文件的播放过程的上下文。视频文件解封装之后的各种信息基本上都放在了这个结构体里面，播放一个视频的时候，一开始就需要做一个avformat_alloc_context的操作去申请整个播放模块的上下文所需要的内存。

这个是可以往里面加字段的，如果要给视频文件添加信息，就可以往这个结构体里面加。

这个结构体也可以视为ffmpeg视角里面的“视频”本身。

AVCodecContext：编解码器的上下文，编解码器类型，视频size，音频通道数和采样率等。

AVCodec：音视频编解码器

编解码器和流是绑定的，会通过分析流的结构，ffmpeg构造一个适合这个流的编解码器。

AVStream：音视频流

这个就是流，同时这个结构体里有很多相关的信息。

AVPacket：音视频编码数据

流里面拿出来的就是这个，这就是编号码的帧

AVFrame：音视频解码数据（原始数据）

packet被解析出来，就是这个东西了，这个里面存的东西就是大家在屏幕上看到的了。

流程：

创建 av format context

打开文件，解封装，找到流的信息，拿到要播放的流的索引。

根据流的信息，选择decoder。创建codec context，开启 codec

codec 从流里面一个 packet 一个 packet 的拿东西，读到 frame 里面。

render 从 frame 里面拿出来数据，渲染到 surface 上。

加入 buffer。

read frame 的内容放在一个 buffer 里面。

晚上回去总结一下这里的坑

了解一下每一步做了什么。

大部分地方的坑都和音视频没什么关系
